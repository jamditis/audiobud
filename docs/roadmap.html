Roadmap

Where AudioBud is headed.

Themes and status, not dates. Each milestone links to its issues on GitHub. Near-term versions carry firm issue lists; the later feature bets are directional, and v1.0.0 is a quality gate.

v0.3.0 — personalization & packaging

in progress

Finalize the on-device learned dictionary so AudioBud adapts to your vocabulary, and tighten the Windows installer.

View milestone →

v0.3.1 — critical stability patch

planned

Default-engine data-loss and crash fixes: history auto-delete, non-ASCII profile paths, clipboard contents, and a watchdog for a stuck transcription.

View milestone →

v0.4.0 — signed & distributable

planned

A signed installer with no SmartScreen warning, the updater repointed to AudioBud so automatic updates work again, and a self-contained install.

View milestone →

v0.5.0 — stability & reliability

planned

The remaining inherited reliability bugs and a full accessibility pass across the settings, model picker, and dialogs.

View milestone →

v0.6.0 — on-device AI cleanup

planned

Optional transcript cleanup with a local LLM — never the cloud — plus cheaper accuracy wins, the long-audio fix, and a real-time preview.

View milestone →

v0.7.0 — interaction & personality

planned

Voice-driven editing commands, an in-app tutorial, and more of the frog mascot's personality.

View milestone →

v1.0.0 — cross-platform & stable

planned

Validate the macOS and Linux builds, finish the remaining hardening, and complete the documentation.

View milestone →
